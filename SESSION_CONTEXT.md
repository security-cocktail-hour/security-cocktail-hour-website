## Episode Content Structure

### Episode Front Matter Format

```yaml
---
title: "Episode Title"
date: YYYY-MM-DD
draft: false
guest: "Guest Name"
category: "Category"
duration: "HH:MM:SS"
image: "/images/episodes/episode-XXX.jpg"
description: >-
  Episode description (uses YAML block scalar for quotes)
platforms:
  youtube: "URL"
  spotify: "URL"
  apple: "URL"
  amazon: "URL"
guest_bio: "Bio text"
---
```

### Episode File Naming Convention

```
episode-[number]-[title-slug].md
```
- Number: Two digits with leading zero (01, 02, ... 66)
- Title slug: Lowercase, hyphenated, max 80 chars total filename

### Episode Content Body Format

After the front matter, the episode markdown should contain:

1. **Opening Paragraph** - Full description of the episode
2. **Episode Timestamps** - Use bulleted list format:
```markdown
## Episode Timestamps

- 00:00 Intro
- 00:38 Welcome Guest Name
- 02:42 Topic description
```

3. **Topics Discussed** - Use bulleted list format (no hashtags):
```markdown
## Topics Discussed

- AI
- Cybersecurity
- CISO
```

**Important**: Use bullet points (dashes) for both timestamps and topics. Do NOT use hashtags.

### Episode Image Format

- **File Location**: `/static/images/episodes/episode-XXX.jpg`
- **Format**: JPG preferred (optimized for web)
- **Naming**: `episode-XXX.jpg` where XXX is the episode number with leading zeros
- Missing images automatically display gradient fallback placeholders

## Git & Netlify Workflow

### Repository Information

**GitHub Repository**: https://github.com/security-cocktail-hour/security-cocktail-hour-website
**Branch**: `main`
**Remote**: `origin`

### Making Changes & Deploying

Netlify is configured for **automatic deployments**. Every push to the `main` branch triggers a new build and deployment.

**Workflow:**

1. **Make changes locally**:
   ```bash
   cd "/Users/joe/Library/CloudStorage/Dropbox/Security Cocktail Hour/website/redesign 2025-10/security-cocktail-hour-website"
   hugo server -D
   # View at http://localhost:1313/
   ```

2. **Test changes locally** - Browse at http://localhost:1313/

3. **Commit and push changes**:
   ```bash
   git add -A
   git commit -m "Your descriptive commit message"
   git push
   ```

4. **Automatic deployment** - Netlify detects push, builds site, deploys to staging (~1-2 minutes)

5. **Verify deployment** - Check https://app.netlify.com/

### Netlify Build Configuration

**File**: `netlify.toml`

```toml
[build]
  command = "hugo --minify"
  publish = "public"

[build.environment]
  HUGO_VERSION = "0.151.0"
  HUGO_ENV = "production"
```
